# Product Copy And Conversion Analysis

## Copy Strategy

The copy is outcome-first. It does not lead with model details or AI jargon; it leads with the user's pain: videos take time to watch, and teams need useful conclusions, subtitles, and reusable insights.

## Messaging Principles

| Principle | How It Appears | Why It Works |
|---|---|---|
| Time saved is the core benefit | Hero, metrics, use-case cards | Makes value obvious to busy operators |
| One link creates many outputs | Hero preview and feature cards | Reduces perceived complexity |
| Output is structured, not vague | Result showcase and feature checklists | Builds trust in usefulness |
| Audience segmentation is explicit | Creator, brand, MCN, ecommerce cards | Lets visitors self-identify quickly |
| Platform segmentation supports SEO | Douyin, Xiaohongshu, Bilibili solution cards | Captures platform-specific intent |
| Developer integration expands maturity | MCP/API section and links | Appeals to power users and batch workflows |
| Registration has an incentive | New-user credit messaging | Reduces signup resistance |

## Problem Statement

Verified theme: manually watching and extracting value from short videos is slow. The site positions the product as a faster way to summarize, transcribe, analyze, and archive videos.

## Promised Outcome

The promised outcome is a short, structured, reusable analysis from a video link. It includes summary, subtitles, insights, archive/history, download URL extraction, and API/MCP access.

## Technical Messaging

Technical concepts are kept near the developer path. The homepage mentions API/MCP as a feature card and preview link, but does not let developer copy overwhelm the creator/operator pitch.

## CTA Pattern

| CTA Type | Placement | Role |
|---|---|---|
| Start analysis | Hero and preview card | Primary conversion |
| Get download URL | Hero and tools route | Lower-friction alternate workflow |
| Guide | Hero and guide route | Reduces uncertainty |
| Register | Header/mobile menu | Account conversion |
| Platform solution links | Solution cards | SEO/deeper exploration |
| MCP docs | Hero preview and nav | Developer conversion |

CTA wording is concise and action-oriented. It favors verbs and task names over abstract marketing language.

## Conversion Flow

```text
Visitor lands
-> Understands product promise in hero
-> Sees simulated one-link workflow
-> Notices supported platforms and benefit metrics
-> Reviews public examples
-> Scans feature/audience/platform sections
-> Reads FAQ
-> Clicks analysis/download/register
-> Logs in or registers
-> Uses analysis tool or download tool
-> Returns through task history / public examples / integrations
```

## Conversion Strengths

1. First viewport answers what the product does and how to start.
2. Public examples make output tangible.
3. Multiple CTAs support different readiness levels.
4. Header keeps registration visible without dominating.
5. Developer route suggests extensibility and seriousness.

## Conversion Risks

1. The main CTA redirects to login for unauthenticated users, so the "try now" feeling is interrupted.
2. The homepage ends after FAQ without a final CTA or footer.
3. The mobile page is long, which may reduce completion.
4. Some feature cards look interactive but do not provide deeper links.
5. The public task examples include long, specific content that may distract from the product value.
