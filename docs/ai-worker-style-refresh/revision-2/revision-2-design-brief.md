# Revision 2 Design Brief

## Why Revision 1 Failed Visually

Revision 1 preserved the internal cockpit structure too closely. The homepage still read as an equal-box operational dashboard with compact cards, compressed hierarchy, and a console-led utility feel. Automated checks passed, but the first visual impression did not communicate a modern AI Worker product.

## Productized Workbench Definition

Productized Workbench means the homepage remains an operational PR Creator tool while gaining product-level framing. The first screen should make the ZTE / AI Worker identity obvious, present a confident headline and workflow promise, and place the real creation controls inside a polished work panel above the fold.

## Required Layout Changes

The homepage should move away from the current four equal cards. The new first screen should use a hero/workbench composition:

- A product identity and headline area with workflow/status chips.
- A primary PR Creator work panel that groups upload, validation, scope, site input, and create job controls.
- A supporting result/status panel that keeps download readiness and live state visible without dominating the first impression.
- A lower operational layer for AI Chatbox and Worker Console.

## Operational Control Preservation

The refresh must preserve all existing workflow capability:

- Upload source file.
- Run validation.
- Select site mode.
- Select task type.
- Enter site codes.
- Create job.
- Track job progress.
- Download ZIP results.
- Ask AI follow-up questions.
- View worker console output.

Only the layout and visual hierarchy should change.

## Difference From Old Cockpit

The old cockpit used compact equal cards and viewport locking, which made the product feel like an internal admin console. Revision 2 should use spacious SaaS rhythm, stronger headline hierarchy, clear primary and secondary actions, layered depth, and a workbench panel that feels designed around the PR Creator journey.

## Human Visual Acceptance Criteria

Human acceptance should judge whether the homepage first screen clearly looks transformed from Revision 1 and visually communicates a modern AI product workbench. Passing tests are required, but they are not sufficient. The user must explicitly accept the visual result before Revision 2 can be marked complete.
